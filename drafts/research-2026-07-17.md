# Research log — 2026-07-17

Real sources found via web search. All links verified as returned by search; content summaries paraphrased, no fabricated numbers.

## 1. AGENTS.md is now the cross-tool standard
As of mid-2026, AGENTS.md is read natively by Claude Code, OpenAI Codex CLI, Cursor, Aider, Devin, Sourcegraph Amp, Google Jules, Zed AI, Continue, Roo Code, Factory Droids, GitHub Copilot, Gemini CLI, Windsurf, and Amazon Q. Donated to the Agentic AI Foundation (Linux Foundation) in Dec 2025.
- https://agents.md/
- https://developers.openai.com/codex/guides/agents-md
- https://blog.buildbetter.ai/agents-md-complete-guide-for-engineering-teams-in-2026/

**Signal for agentsmd:** the standard exists; the *manager* for it doesn't.

## 2. The duplicate-rules pain is well-articulated
Multiple 2026 blog posts explicitly describe the "N copies of the truth" problem when tools each want their own file (`CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md`, `.windsurfrules`, `AGENTS.md`). Current workarounds: symlinks, hard links, or hand-rolled sync scripts.
- https://pub.towardsai.net/your-ai-agent-rules-one-source-of-truth-for-cursor-claude-code-and-every-tool-youll-adopt-next-cd7f1353dbbc
- https://www.concret.io/blog/sync-coding-standards-across-cursor-agentforce-vibes-claude
- https://www.deployhq.com/blog/ai-coding-config-files-guide

**Signal for agentsmd:** users are literally writing bash scripts and pre-commit hooks to do what `agentsmd sync` should do out of the box.

## 3. Cursor and Claude Code split behavior
Cursor reads `.cursor/rules/*.mdc` and `AGENTS.md` (not `CLAUDE.md`). Claude Code reads `CLAUDE.md` only, not `AGENTS.md` natively. This asymmetry keeps the manual-sync problem alive even with AGENTS.md adoption.
- https://techsy.io/en/blog/cursor-rules-vs-claude-md
- https://paddo.dev/blog/claude-rules-path-specific-native/

**Signal for agentsmd:** even the "standard" doesn't remove the tool-specific-file need. Sync remains valuable.

## 4. Recommended AGENTS.md sections (for `init` scaffolding)
Spec is plain Markdown, no required fields, but consensus recommended sections: overview, build/test commands, code style, testing, security, commit/PR rules. Use this as the default `agentsmd init` template.
- https://www.morphllm.com/agents-md-guide
- https://www.augmentcode.com/guides/how-to-build-agents-md

**Signal for agentsmd:** `init` should scaffold with these six sections and prompts.

## 5. Name-collision risk
`github.com/agentsmd/agents.md` is an existing org/repo — it's the canonical AGENTS.md spec home, not a competing tool. But there is a risk of user confusion. Consider `agentsmd-cli` as npm/binary name and keep the repo name.
- https://github.com/agentsmd/agents.md

**Signal for agentsmd:** DECISION NEEDED — npm package name to avoid collision + brand clarity.
