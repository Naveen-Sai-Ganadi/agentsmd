## Signals worth chasing

1. **Monorepo CLAUDE.md sprawl** — repeated complaint that each package gets its own `CLAUDE.md` that diverges from the root. Users want an inheritance/merge model. Direct hit for `sync` (per-package overrides on top of a root `AGENTS.md`).
2. **AGENTS.md ↔ CLAUDE.md drift** — several teams reporting they set up both, then only remember to update one. This is the *exact* problem the sync command shipped today addresses; validates the wedge.
3. **Cursor "Rules for AI" nesting** — Cursor now supports `.cursor/rules/*.mdc` per-directory. Any sync tool needs to understand nested rules, not just the legacy root `.cursorrules`. Log for v0.2.
4. **Copilot workspace instructions** — GitHub extended `.github/copilot-instructions.md` to also read `.github/instructions/*.instructions.md`. Same nested-rules pattern as Cursor. Log for v0.2.
5. **Codex CLI reads AGENTS.md natively** — reinforces the "AGENTS.md as source of truth" bet. Good line for the launch post.

## Implications for roadmap
- v0.1.0 stays flat-file. Ship it, get eyeballs.
- v0.2.0 must handle nested rules for Cursor AND Copilot — same shape, different directory. Add a `NestedRulesEngine` shared by both.
- `lint` should flag "you have both AGENTS.md and CLAUDE.md with different content" as a P0 finding — that's the drift users are already feeling.
