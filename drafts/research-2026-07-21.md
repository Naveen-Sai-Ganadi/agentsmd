## Prior art / competitors (be honest, not defensive)

- **spxrogers/agentsync** — GitHub tool that syncs Claude Code, OpenCode, Codex, and more from one canonical source. Direct competitor; must read their README before next design decision. https://github.com/spxrogers/agentsync
- **amtiYo/agents** — `.agents` folder as source of truth, syncs MCP servers + skills + instructions across Codex, Claude Code, Gemini CLI, Cursor, Copilot, Antigravity. Broader scope than us (skills + MCP). https://github.com/amtiYo/agents
- Neither project (from titles alone) targets the *lint/audit* dimension. That's our wedge — `agentsmd lint` / `agentsmd audit` (Roadmap #5) is the differentiator to double down on rather than another sync CLI.

## Implication for roadmap

- **Reprioritise:** consider bumping `lint` + `audit` (roadmap #5) ahead of `check` CI mode (#6) so we ship a differentiated capability before v0.1.0.
- **Investigate** spxrogers/agentsync and amtiYo/agents in depth tomorrow — feature-by-feature diff, note whether our banner + dry-run defaults are actually novel or table stakes.
- **Positioning:** rewrite README tagline from "sync tool" → "the linter and auditor for AGENTS.md (with sync built in)". Do not do this until we've read the competitors so we don't overclaim.
